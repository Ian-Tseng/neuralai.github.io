## Learning discipline

- At every meaningful update, capture durable lessons with the project learning tools.
- Log only reusable knowledge: root causes, verified constraints, user preferences, workflow improvements, and mistakes that would change future behavior.
- Do not log routine status, speculation, or one-off details that will not help a later agent.
- Write each learning with a short key, concrete evidence, confidence level, source, and relevant file paths when applicable.
- Before starting substantial work, search prior learnings and apply any matching high-confidence guidance.

## Documentation and update-log discipline

- At every meaningful update, update the related `.md` file with the current decision, change, source of truth, or next-step context.
- If no related `.md` exists, create a focused new Markdown file with a clear name rather than hiding durable project context in chat history.
- Keep Markdown updates concise, dated when useful, and directly tied to project reality: what changed, why it changed, what source was used, and what remains uncertain.
- Do not duplicate the same note across many files. Prefer one canonical location and link or reference it from other documents when needed.
- Treat Markdown as the handoff layer for future agents: enough context to continue work without rereading the whole conversation, but no routine status noise.

## Inconsistency discipline

- At every meaningful update, check for inconsistencies before reporting the work as done.
- Look for duplicate assets, stale drafts, mismatched version names, temp or lock files, broken links, missing referenced files, and claims that conflict with available source material.
- Prefer objective checks first: file hashes, file sizes, modification dates, link validation, and direct source inspection.
- Report only actionable inconsistencies. Do not block delivery on harmless archives or intentional historical versions unless they create ambiguity for the next agent.
- When fixing inconsistencies, preserve the newest reliable source and avoid deleting project artifacts unless the user explicitly approves.
